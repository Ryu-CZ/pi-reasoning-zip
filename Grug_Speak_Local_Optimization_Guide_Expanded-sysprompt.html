1. The Core Idea: Copy OpenAI Token Discipline Locally

OpenAI models achieve excellent token efficiency through internal “Grug-speak” — terse, keyword-heavy internal reasoning that never reaches the user. On open models we can approximate this with two complementary techniques:

  1. Strict system prompt that forces concise, structured output (the “Grug” persona)
  2. Engine-level controls (MTP speculative decoding + generation caps + thinking bypass)

Together they deliver significantly fewer output tokens and higher tokens-per-second on your RTX 4090.

Verified model: Qwen3.6-27B-MTP (dense) with native Multi-Token Prediction heads — exactly the model this guide targets.

2. The Grug System Prompt (Copy-Paste Ready)

Important: Always start with the official Qwen base prompt for best performance:

You are Qwen, created by Alibaba Cloud. You are a helpful assistant.

Then append your Grug instructions:

You are Qwen, created by Alibaba Cloud. You are a helpful assistant.

You are Grug. Grug code simple. Grug output clean. 
Save token, save world. Follow rules strictly:

1. No "Sure, I can help with that!" No conversational fluff or polite filler.
2. Do not repeat user question. Do not explain obvious things.
3. If user ask for code, give ONLY code. Use code blocks. No intro, no outro text.
4. If text needed, use short bullet points or broken sentences. Avoid big words. 
5. Cut explanations unless user explicitly ask "why". 
6. Keep markdown structure minimal.

Think hard, output few tokens. Start now.

How the Prompt Works

  • Persona lock — “You are Grug” sets a strong, consistent voice that Qwen3.6 obeys very well.
  • Explicit anti-fluff rules — directly attacks the most common sources of wasted tokens.
  • Code-only rule — extremely effective for coding agents.
  • Final instruction — “Think hard, output few tokens” encourages internal reasoning while keeping the visible answer short.

3. Where Does the System Prompt Live?

llama-server — Per-Request System Prompt (Current Correct Method)

Current llama-server does not accept --system-prompt as a startup flag. Put the Grug prompt into each API request instead:

  • OpenAI Chat Completions: first item in messages is {"role":"system"}.
  • OpenAI Responses: use the top-level instructions field.
  • Anthropic Messages: use the top-level system field.

Pros: Works with current llama-server, gives each agent clean control, and lets normal / concise modes coexist.
Cons: Every client must send the prompt or configure it in its own template.

llama-cli — Startup System Prompt

The CLI frontend still supports a launch-time system prompt:

./llama-cli \
  -m /models/Qwen3.6-27B-MTP-Q4_K_M.gguf \
  --system-prompt "You are Qwen, created by Alibaba Cloud. You are a helpful assistant.

You are Grug. Grug code simple. Grug output clean. 
Save token, save world. Follow rules strictly: [paste full Grug rules here]"

Recommended for Agents

Pass the system message as the first message with role: "system" in every API call. This is the standard OpenAI-compatible pattern.

This is the best choice for “Pi Agent” and “Hermes Agent”.

Answer to your question:
You should edit your Pi Agent and Hermes Agent to include the Grug system prompt in their configuration / prompt template.
Do not use server-level --system-prompt; current llama-server does not expose that flag. Put the prompt in each agent request.

Key Flags Explained

Flag Purpose Recommended Value
--spec-type draft-mtp Enable native Multi-Token Prediction Required for speed
--spec-draft-n-max 2 How many future tokens to speculate 2 (sweet spot) or 3
--reasoning off Disable Qwen <think> mode Essential for Grug brevity
--no-reasoning-preserve Do not preserve reasoning traces in history Good default when thinking is off
-n 512 Hard max output tokens 256–768 depending on task
--reasoning-budget 0 Completely disable reasoning budget 0 when thinking is off

5. How to Use in Your Agents (Pi & Hermes)

General Pattern (OpenAI-compatible endpoint)

Most agents that talk to llama-server use the /v1/chat/completions endpoint. You control the system prompt in the messages array.

Example: Python (openai client)

from openai import OpenAI

client = OpenAI(base_url="http://localhost:8080/v1", api_key="sk-no-key")

GRUG_PROMPT = """You are Grug. Grug code simple. Grug output clean. 
Save token, save world. Follow rules strictly:
1. No "Sure, I can help with that!" No conversational fluff...
Think hard, output few tokens. Start now."""

response = client.chat.completions.create(
    model="grug-qwen",
    messages=[
        {"role": "system", "content": GRUG_PROMPT},
        {"role": "user", "content": "Write a function that sorts a list of dicts by the 'score' key."}
    ],
    max_tokens=512,
    temperature=0.65
)

print(response.choices[0].message.content)

Example: curl

curl http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "grug-qwen",
    "messages": [
      {"role": "system", "content": "You are Grug. ... Start now."},
      {"role": "user", "content": "Explain how MTP works in one sentence."}
    ],
    "max_tokens": 256,
    "temperature": 0.6
  }'

Pi Agent & Hermes Agent Specific Advice

  1. Find where your agent defines its system message or default prompt (usually a YAML/JSON config, a Python constant, or a prompt template file).
  2. Replace or prepend the existing system prompt with the full Grug prompt.
  3. If the agent supports “preset” or “personality” switching, create a new preset called grug or concise.
  4. Keep the system prompt as the very first message in every conversation turn (standard behavior for most frameworks).
  5. Test with a simple coding request — you should see dramatically shorter, cleaner output immediately.
Pro tip: Store the Grug prompt in a separate grug_system.txt file and load it in your agent code. This makes future tweaks trivial.

8. Troubleshooting

System prompt not applied

For llama-server, the system prompt is request data, not a launch flag. Check the API shape:

  • Chat Completions: first messages entry uses {"role":"system"}.
  • Responses: top-level instructions contains the Grug prompt.
  • Anthropic Messages: top-level system contains the Grug prompt.

Model still produces long answers

  • Make sure --reasoning off is set on llama-server.
  • Lower -n (e.g. 256 or 384).
  • Re-check that the Grug system prompt is actually the first message being sent.

MTP not giving speedup

Confirm you are using a model with -MTP- in the filename and the latest llama.cpp build that supports draft-mtp.
